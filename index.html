tiny-httpd

A static-file HTTP/1.1 server written from scratch in C — no libevent, no libuv, just POSIX sockets and a hand-rolled request parser. The core is a single-threaded epoll event loop driving non-blocking I/O, so one slow or half-sent request never stalls any other connection.

  • Non-blocking epoll event loop, HTTP/1.1 keep-alive, sendfile() zero-copy
  • SO_REUSEPORT multi-worker processes, idle-connection timeouts, access log
  • Live /dashboard over Server-Sent Events, with a "Generate load" button that makes the concurrency visible
  • One-command Docker demo (make demo), wrk benchmark table in the README
  • Path-traversal-hardened static file serving; clean under ASan/UBSan/valgrind in CI

flashcards-api

A REST API for studying flashcards with spaced repetition — decks, cards, and a review endpoint that schedules each card's next appearance with an SM-2-lite algorithm, the same family Anki uses.

  • FastAPI + SQLAlchemy + SQLite, zero setup, no database server to run
  • The scheduler is a pure function, unit-tested independently of the API and the database
  • Deck/card CRUD, due-today review queue, /stats aggregate counts
  • Auto-generated interactive docs at /docs (Swagger UI)
  • 19 pytest tests, GitHub Actions CI
